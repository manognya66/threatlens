What I Learned

I configured inter-VLAN routing using a single router interface with sub-interfaces — the "Router on a Stick" method. The coolest part: two computers on different VLANs (completely isolated at Layer 2) could ping each other through the router.

Before this lab, I didn't fully understand why we need a router to communicate between VLANs. Now I get it: VLANs are separate broadcast domains, so they need Layer 3 routing to talk to each other.

What is Inter-VLAN Routing?

Inter-VLAN routing allows devices on different VLANs to communicate with each other. Since VLANs are isolated broadcast domains at Layer 2, you need a Layer 3 device (router or Layer 3 switch) to route traffic between them.

Why Do We Need Inter-VLAN Routing?

VLANs segment a network into separate broadcast domains. Without inter-VLAN routing:

  • VLAN 10 cannot talk to VLAN 20
  • Each VLAN is completely isolated
  • No communication between departments, even on the same physical switch

What is "Router on a Stick"?

Router on a Stick is a method where a single router interface connects to a switch trunk port and uses sub-interfaces to route between VLANs. Each sub-interface represents a different VLAN and acts as the default gateway for that VLAN.

Why "Router on a Stick"? The name comes from the visual: one physical router interface (the "stick") handles all VLANs through logical sub-interfaces. It's like plugging all VLANs into one router port.

Key Concepts

Term What It Means
Trunk Port Switch port that carries traffic for multiple VLANs (tagged with VLAN IDs)
Access Port Switch port that carries traffic for only one VLAN (untagged)
Sub-interface Logical interface on a router (e.g., Fa0/0.10, Fa0/0.20)
802.1Q (dot1q) IEEE standard for VLAN tagging on trunk links
Encapsulation Adding VLAN tags to frames so they can be identified on trunk links
Default Gateway Router IP address that devices use to reach other networks

Inter-VLAN Routing Methods

Method How It Works Pros / Cons
Router on a Stick One router interface, multiple sub-interfaces ✅ Cost-effective
❌ Single point of failure
Layer 3 Switch Switch with routing capabilities (SVI) ✅ Faster routing
❌ More expensive
Separate Router Interfaces One physical interface per VLAN ✅ Simple
❌ Wastes router ports

Configuration Steps

Step 1 — Configure Switch A

Cisco IOS — Switch A 
enable
configure terminal

! Create VLANs
vlan 10
 name Sales
 exit
vlan 20
 name Engineering
 exit

! Configure trunk port to router
interface fastethernet0/1
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport trunk allowed vlan 10,20
 no shutdown
 exit

! Configure access port for Computer A (VLAN 10)
interface fastethernet0/2
 switchport mode access
 switchport access vlan 10
 no shutdown
 exit

! Configure access port for Computer B (VLAN 20)
interface fastethernet0/3
 switchport mode access
 switchport access vlan 20
 no shutdown
 exit

! Save configuration
end
write memory

! Verify configuration
show vlan brief
show interfaces trunk
show interfaces fastethernet0/1 switchport
Trunk Encapsulation: The command switchport trunk encapsulation dot1q is required on some switches (like Catalyst 3560) before setting trunk mode. It tells the switch to use 802.1Q tagging.

Step 2 — Configure Router A (Router on a Stick)

Cisco IOS — Router A 
enable
configure terminal

! Create sub-interface for VLAN 10
interface fastethernet0/0.10
 encapsulation dot1q 10
 ip address 192.168.10.254 255.255.255.0
 no shutdown
 exit

! Create sub-interface for VLAN 20
interface fastethernet0/0.20
 encapsulation dot1q 20
 ip address 192.168.20.254 255.255.255.0
 no shutdown
 exit

! Enable the physical interface
interface fastethernet0/0
 no shutdown
 exit

! Save configuration
end
write memory

! Verify configuration
show ip interface brief
show ip route
show interfaces fastethernet0/0.10
show interfaces fastethernet0/0.20
Important: You must enable the physical interface (interface fa0/0 → no shutdown) or the sub-interfaces won't work, even if they're configured correctly.

Step 3 — Configure End Devices

Computer A (VLAN 10)

IP Address 192.168.10.1
Subnet Mask 255.255.255.0
Default Gateway 192.168.10.254

Computer B (VLAN 20)

IP Address 192.168.20.1
Subnet Mask 255.255.255.0
Default Gateway 192.168.20.254

Step 4 — Testing Connectivity

From Computer A 
! Ping default gateway (should work)
ping 192.168.10.254

! Ping Computer B in different VLAN (should work via router)
ping 192.168.20.1

! Trace route to see path
tracert 192.168.20.1
After configuration, I pinged from Computer A (192.168.10.1) to Computer B (192.168.20.1) and it worked! The tracert showed the packet going through 192.168.10.254 (the router) first, proving inter-VLAN routing was working.

Key Takeaways

What I Understood

  • VLANs are Layer 2 constructs — they can't route traffic by themselves
  • Router sub-interfaces act as default gateways for each VLAN
  • The trunk port carries tagged traffic for multiple VLANs using 802.1Q
  • Access ports are untagged — devices don't know they're on a VLAN
  • You must enable the physical interface for sub-interfaces to work
  • Each sub-interface needs encapsulation dot1q [vlan-id]
  • Without inter-VLAN routing, VLAN 10 and VLAN 20 can't communicate

Common Mistakes I Made

  • Forgot to enable Fa0/0: Sub-interfaces were configured but didn't work until I did no shutdown on the physical interface
  • Wrong VLAN on trunk: Initially forgot to add VLAN 20 to allowed VLANs on trunk — Computer B couldn't reach the router
  • Mismatched encapsulation: Put dot1q 10 on the wrong sub-interface — took me a while to figure out why VLAN 10 had VLAN 20's IP range

When to Use Router on a Stick

Good For Not Good For
Small networks (2-5 VLANs) Large enterprise networks
Budget-conscious setups High-traffic environments
Lab/learning environments When Layer 3 switch is available
When you have limited router ports Networks requiring redundancy
