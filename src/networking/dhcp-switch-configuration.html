What I Learned

I configured a multilayer switch to act as a DHCP server, automatically assigning IP addresses to computers in VLAN 10. Computer A received an IP address dynamically without any manual configuration — just set it to DHCP and it worked.

The key insight: switches can be DHCP servers, not just routers. This is useful when you have a Layer 3 switch handling both routing and IP address distribution for a VLAN.

What is DHCP?

DHCP (Dynamic Host Configuration Protocol) automatically assigns IP addresses, subnet masks, default gateways, and DNS servers to devices on a network. Instead of manually configuring every computer, DHCP handles it automatically.

DHCP Process (DORA)

Step Message What Happens
1. Discover DHCP Discover Client broadcasts "I need an IP address"
2. Offer DHCP Offer Server offers an available IP address
3. Request DHCP Request Client requests the offered IP
4. Acknowledge DHCP ACK Server confirms and assigns the IP

DHCP vs Static IP

Feature DHCP (Dynamic) Static IP
Configuration Automatic Manual
Management Centralized Per-device
IP Address Changes Yes (lease-based) No (permanent)
Best For End-user devices, laptops, phones Servers, printers, network devices
Scalability Excellent Poor
Why Use DHCP? Imagine managing 100 computers manually — every IP, subnet mask, gateway needs manual entry. DHCP does this automatically and prevents IP conflicts.

My Lab Topology

One multilayer switch acting as DHCP server for VLAN 10, with Computer A connected and receiving an IP address automatically.

Network Diagram

Topology:
  • Computer A (VLAN 10) → Switch Fa0/1
  • Switch A = DHCP Server for VLAN 10
  • DHCP Pool: systechvlan10
  • Network: 192.168.10.0/24

DHCP Configuration Details

Parameter Value
DHCP Pool Name systechvlan10
Network 192.168.10.0
Subnet Mask 255.255.255.0 (/24)
Default Gateway 192.168.10.254
Excluded Address 192.168.10.254 (gateway)
Available Range 192.168.10.1 - 192.168.10.253

Expected Behavior

DHCP Process:
  1. Computer A boots up and requests IP via DHCP
  2. Switch offers an IP from the pool (e.g., 192.168.10.1)
  3. Computer A accepts the IP
  4. Switch assigns: IP, subnet mask, default gateway
  5. Computer A can now communicate on the network

Configuration Steps

Step 1 — Create VLAN 10

Cisco IOS — Switch A 
enable
configure terminal

! Create VLAN 10
vlan 10
 exit

! Configure access port for Computer A
interface fastethernet0/1
 switchport mode access
 switchport access vlan 10
 no shutdown
 exit

Step 2 — Configure DHCP Pool

Create DHCP Pool 
! Create DHCP pool named "systechvlan10"
ip dhcp pool systechvlan10
 network 192.168.10.0 255.255.255.0
 default-router 192.168.10.254
 exit
DHCP Pool Commands:
  • network — Defines the subnet and mask
  • default-router — Sets the gateway for clients
  • Pool name can be anything descriptive

Step 3 — Exclude Gateway Address

Exclude Addresses 
! Exclude gateway IP from DHCP pool
! (Prevents DHCP from assigning .254 to a client)
ip dhcp excluded-address 192.168.10.254

! Save configuration
end
write memory
Important: Always exclude gateway addresses, server IPs, and any static IPs from the DHCP pool. You can also exclude ranges: ip dhcp excluded-address 192.168.10.1 192.168.10.50

Step 4 — Verify DHCP Configuration

Verification Commands 
! Show DHCP bindings (assigned IPs)
show ip dhcp binding

! Show DHCP pool configuration
show ip dhcp pool

! Show excluded addresses
show ip dhcp excluded-address

! Debug DHCP server (see DHCP process in real-time)
debug ip dhcp server packet

! Turn off debug
undebug all

Understanding show ip dhcp binding Output

Sample Output 
Switch-A# show ip dhcp binding

                 Hardware address

Step 5 — Configure Computer A for DHCP

1

Windows: Set to Obtain IP Automatically

Control Panel → Network Connections → Properties → TCP/IPv4 → Select "Obtain an IP address automatically"

2

Release and Renew IP

Open Command Prompt: ipconfig /release then ipconfig /renew

3

Verify IP Assignment

Run ipconfig /all and check that IP is in 192.168.10.x range

After running ipconfig /renew, Computer A received 192.168.10.1 automatically! I verified with show ip dhcp binding on the switch and saw the lease active.

Key Takeaways

What I Understood

  • Multilayer switches can act as DHCP servers
  • DHCP follows DORA process: Discover, Offer, Request, Acknowledge
  • ip dhcp pool creates a named DHCP pool
  • network defines the subnet to assign from
  • default-router sets the gateway for clients
  • Always exclude gateway and server IPs from DHCP pool
  • show ip dhcp binding shows active leases

DHCP Pool Commands Summary

Command What It Does
ip dhcp pool [name] Create DHCP pool with a name
network [ip] [mask] Define network subnet
default-router [ip] Set default gateway for clients
dns-server [ip] Set DNS server (optional)
lease [days] Set lease duration (default: 1 day)
ip dhcp excluded-address [ip] Exclude specific IPs from pool

DHCP Best Practices

Practice Why
Exclude gateway addresses Prevent IP conflicts
Reserve IPs for servers Servers need static or reserved IPs
Use descriptive pool names Easier management (e.g., "VLAN10_Pool")
Document excluded ranges Know which IPs are reserved
Monitor DHCP bindings Detect pool exhaustion early

Common Mistakes

  • Didn't exclude gateway: DHCP assigned .254 to a computer, causing gateway conflict
  • Wrong subnet mask: Used /25 instead of /24 — clients got wrong subnet configuration
  • Forgot ipconfig /renew: Computer didn't get IP until I manually renewed the lease
