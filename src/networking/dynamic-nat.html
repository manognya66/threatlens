What I Learned

After understanding Static NAT, I moved on to Dynamic NAT. The key difference I noticed: instead of a fixed one-to-one mapping, the router automatically picks an available IP from a pool whenever a device needs to communicate outside.

Configuring the ACL and NAT pool together was the interesting part — it made me understand how routers decide which traffic to translate and which IP to assign.

What is Dynamic NAT?

Dynamic NAT maps private IPs to a pool of public IPs. Unlike Static NAT, the mapping isn't permanent — the router assigns one temporarily and releases it when the session ends.

Think of it like a shared visitor pass. You take one when you need to go outside, and return it when you're done. The next person reuses it.

How It Works

1

ACL Check

Router checks if the source IP is permitted for NAT

2

Pool Assignment

Router picks a free public IP from the NAT pool

3

Translation

Private IP is temporarily mapped to the chosen public IP

4

Release

After the session ends, the public IP returns to the pool

NAT Pool I Configured

Public IP pool on Router A: 200.100.100.1 — 200.100.100.5

Any device in the 10.0.0.0/8 network gets one of these IPs temporarily assigned when communicating outward.

Configuration Commands

Router A

Cisco IOS 
enable
conf t
access-list 1 permit 10.0.0.0 0.255.255.255
ip nat pool master 200.100.100.1 200.100.100.5 netmask 255.255.255.248
ip nat inside source list 1 pool master overload
interface fastethernet0/0
 ip address 10.0.0.1 255.255.255.0
 ip nat inside
 no shutdown
interface serial0/0/0
 ip address 20.0.0.1 255.255.255.0
 ip nat outside
 no shutdown

Router B

Cisco IOS 
enable
conf t
interface fastethernet0/0
 ip address 192.168.0.1 255.255.255.0
 no shutdown
interface serial0/0/0
 ip address 20.0.0.2 255.255.255.0
 no shutdown

Verification

Cisco IOS 
show ip nat translations
show ip nat statistics
When I ran show ip nat translations, I could see dynamic entries showing which pool IP was currently assigned to my device. Once the session ended, the entry disappeared.

Key Takeaways

What I Understood

  • ACL defines which traffic gets translated
  • NAT pool holds the available public IPs
  • overload keyword enables PAT (multiple devices share one IP)
  • Mappings are temporary — released after session ends
  • Better than Static NAT when you have many devices but few public IPs

Static NAT vs Dynamic NAT

Feature Static NAT Dynamic NAT
Mapping Permanent Temporary
IP Usage One-to-one Pool-based
Use Case Servers Multiple clients
