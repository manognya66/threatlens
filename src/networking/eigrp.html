What I Learned

I set up a three-router EIGRP lab and experimented with manipulating delay values to see how it affects path selection. What surprised me: EIGRP doesn't just count hops like RIP — it actually thinks about the quality of the path.

The moment I changed the delay on one interface and saw the routing table update automatically, the concept finally made sense.

What is EIGRP?

EIGRP (Enhanced Interior Gateway Routing Protocol) is a hybrid routing protocol developed by Cisco. It combines the best of distance-vector and link-state routing.

Key Facts

  • Uses Protocol Number 88
  • Multicast address: 224.0.0.10
  • No periodic updates — only sends updates when topology changes
  • Supports IPv4 and IPv6
  • AD of 90 (internal), 170 (external)

DUAL Algorithm

The brain of EIGRP is DUAL (Diffusing Update Algorithm). It selects the best path (Successor) and keeps a backup ready (Feasible Successor).

When a link fails, EIGRP doesn't recalculate from scratch — it instantly switches to the Feasible Successor. That's why convergence is so fast.

Metric — How Best Path is Chosen

By default, EIGRP uses Bandwidth and Delay to calculate the best path (not hop count). In my lab, I changed the K-values to use only Delay, which made it easier to see how path selection works.

Table What It Stores
Neighbor TableDirectly connected EIGRP routers
Topology TableAll learned routes (successors + backups)
Routing TableBest paths chosen by DUAL

My Lab Topology

Three routers (RA, RB, RC) connected in a triangle. I deliberately set different delay values on serial interfaces to force EIGRP to prefer one path over another.

By changing delay on RB and RC, I could watch EIGRP recalculate and change the preferred route in real time.

Autonomous System Number

All three routers used AS 100. EIGRP neighbors only form when the AS number matches — I confirmed this by testing with mismatched AS numbers first.

Configuration Commands

Router A (RA)

Cisco IOS 
enable
configure terminal
router eigrp 100
 metric weights 0 0 0 1 0 0
 network 10.0.0.0
 network 20.0.0.0
 network 60.0.0.0
metric weights 0 0 0 1 0 0 sets K3=1 so only Delay is used for metric calculation. This makes it easier to experiment with path selection.

Router B (RB) — Set Delay First

Cisco IOS 
configure terminal
interface serial0
 delay 5
interface serial1
 delay 10
router eigrp 100
 network 20.0.0.0
 network 30.0.0.0
 network 40.0.0.0

Router C (RC) — Set Delay First

Cisco IOS 
configure terminal
interface serial0
 delay 5
interface serial1
 delay 3
router eigrp 100
 network 40.0.0.0
 network 50.0.0.0
 network 60.0.0.0

Verification

Cisco IOS 
show ip eigrp neighbors
show ip eigrp topology
show ip route
After changing delay values and running show ip eigrp topology, I could see which path became the Successor and which became the Feasible Successor. Changing delay changed the metric and flipped the preferred route.

Key Takeaways

What I Understood

  • EIGRP uses Bandwidth + Delay by default, not hop count
  • DUAL keeps a backup path ready — failover is instant
  • AS numbers must match for neighbors to form
  • Delay is the easiest metric to manipulate in a lab
  • No periodic updates — only sends when something changes
