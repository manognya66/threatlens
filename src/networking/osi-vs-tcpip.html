What I Learned

I spent time in Packet Tracer trying to understand how the OSI and TCP/IP models actually work when you open a website. Everyone memorizes the layers, but I wanted to see them in action.

The biggest realization: these aren't just theoretical concepts. Every time data moves across a network, it goes through these layers — wrapping and unwrapping information at each step.

The Two Models

OSI (7 Layers) TCP/IP (4 Layers) What Happens
Application
Presentation
Session		 Application User programs, data formatting, encryption
Transport Transport TCP/UDP, ports, reliability
Network Internet IP addresses, routing
Data Link
Physical		 Network Access MAC addresses, physical transmission

OSI = Teaching model (theoretical)
TCP/IP = Practical model (what actually runs on networks)

Real Example: Opening google.com

When I opened https://google.com in my lab browser, here's what I observed happening:

1. DNS Resolution (Application Layer)

Browser asks: "What's the IP address of google.com?"
DNS server replies with the IP.

2. TCP Handshake (Transport Layer)

Three-way handshake begins:
SYN → SYN-ACK → ACK
Port 443 for HTTPS is used.

3. IP Routing (Network Layer)

Router checks: "Is this IP in my network?"
No → Send to default gateway
Source IP and Destination IP added to packet.

4. MAC Addressing (Data Link Layer)

Router needs MAC address of next hop
ARP request: "Who has this IP?"
Switches forward frames using MAC tables.

5. Physical Transmission

Bits travel as electrical signals through cables.

Encapsulation: Each layer adds its own header. Data → Segment → Packet → Frame → Bits

Key Takeaways

What I Understood

  • OSI and TCP/IP aren't competing — they describe the same process differently
  • Every network communication goes through these layers
  • Troubleshooting becomes easier when you know which layer to check
  • Packet Tracer's Simulation Mode is perfect for visualizing this

For Troubleshooting

Now when something breaks, I know where to look:

  • Can't reach website? → Check Application/Transport layer (DNS, ports)
  • Can't ping outside network? → Check Network layer (routing, gateway)
  • Can't communicate with device on same network? → Check Data Link (ARP, MAC)
  • No link lights? → Check Physical layer (cables)
