What I Learned

I configured RIP v2 across three routers connecting Trichy, Coimbatore, and Chennai. The biggest difference I noticed from RIP v1: this version supports VLSM and classless routing, which makes it much more practical for real networks.

Watching routes propagate automatically after enabling RIP v2 on each router was satisfying — no manual static routes needed.

What is RIP v2?

RIP v2 (Routing Information Protocol version 2) is a distance-vector routing protocol that automatically shares routing information between routers. It's an upgrade from RIP v1 with support for VLSM and classless networks.

Common Misconception: The document says "RIPv2 is a Link State Routing Protocol" — this is incorrect. RIP v2 is actually a Distance-Vector routing protocol. Link-state protocols include OSPF and IS-IS.

RIP v2 Characteristics

Property Value
Protocol Type Distance-Vector (NOT Link State)
Administrative Distance 120
Metric Hop Count
Maximum Hop Count 15 (16 = unreachable)
VLSM Support Yes
Classless Routing Yes
Update Timer 30 seconds
Hold-down Timer 180 seconds
Flush Timer 240 seconds

RIP v2 Timers Explained

Timer Duration What It Does
Update 30 seconds Router sends routing updates to neighbors
Hold-down 180 seconds After a route fails, router ignores updates about that route
Flush 240 seconds Remove route from routing table if no updates received

RIP v1 vs RIP v2

Feature RIP v1 RIP v2
VLSM Support No Yes
Classless Routing No (Classful) Yes
Subnet Mask in Updates No Yes
Authentication No Yes
Multicast Address Broadcast (255.255.255.255) Multicast (224.0.0.9)

My Lab Topology

Three routers connected in a line: Trichy → Coimbatore → Chennai. Each router has a LAN and connects to neighbors via serial links.

Router Location Interface IP Address Subnet Mask
Router A Trichy Fa0/0 10.32.0.1 255.224.0.0
S0/0/0 10.16.0.1 255.240.0.0
Router B Coimbatore S0/0/0 10.16.0.2 255.240.0.0
S0/0/1 10.8.0.1 255.248.0.0
Router C Chennai Fa0/0 10.4.0.1 255.252.0.0
S0/0/0 10.8.0.2 255.248.0.0

End Devices

Device IP Address Subnet Mask Default Gateway
PC-1 (Trichy) 10.32.0.2 255.224.0.0 10.32.0.1
PC-2 (Chennai) 10.4.0.2 255.252.0.0 10.4.0.1
Notice the different subnet masks across the network — this is VLSM (Variable Length Subnet Masking). RIP v2 handles this perfectly, while RIP v1 would fail.

Verification Commands

Verification 
! Check IP addresses on all interfaces
show ip interface brief

! View the routing table (should show RIP routes with 'R')
show ip route

! View RIP-specific information
show ip protocols

! Debug RIP updates (use carefully - generates lots of output)
debug ip rip

! Turn off debug
undebug all
After configuring all three routers, I ran show ip route and saw routes marked with "R" (RIP). PC-1 in Trichy could ping PC-2 in Chennai successfully — RIP v2 was working.

Key Takeaways

What I Understood

  • RIP v2 is a Distance-Vector protocol, not Link State
  • version 2 command is critical — without it, router defaults to RIP v1
  • RIP v2 supports VLSM and classless routing (unlike RIP v1)
  • Maximum hop count is 15 — any route 16 hops away is unreachable
  • Updates are sent every 30 seconds automatically
  • Administrative Distance of 120 makes RIP less preferred than EIGRP (90) or OSPF (110)
  • The network command uses classful network addresses

When to Use RIP v2

Good For Not Good For
Small networks (under 15 hops) Large enterprise networks
Simple topologies Complex redundant topologies
Learning routing protocols High-performance requirements
Low-bandwidth links acceptable Where fast convergence is critical
