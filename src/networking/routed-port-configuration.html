What I Learned

I configured a Layer 3 switch port as a routed port, turning it from a switchport into a router interface. The key insight: no switchport converts a switch port to act like a router port, allowing direct IP address assignment and routing between VLANs without needing an external router.

Two computers in different VLANs (Computer A in VLAN 10, Computer B in VLAN 1) could ping each other through the switch itself — the switch was routing between them at Layer 3.

What is a Routed Port?

Routed port is a physical switch port configured to behave like a router interface. Instead of operating at Layer 2 (switching), it operates at Layer 3 (routing) and can have an IP address assigned directly to it.

Routed Port vs Switchport

Feature Switchport (Layer 2) Routed Port (Layer 3)
OSI Layer Layer 2 (Data Link) Layer 3 (Network)
IP Address No (uses VLAN interface) Yes (directly on port)
VLAN Support Yes (access or trunk) No (not associated with VLAN)
Routing No Yes
Use Case Connect end devices Connect to routers or other Layer 3 devices
Configuration Command switchport mode access/trunk no switchport
When to Use Routed Ports: Routed ports are used when connecting a Layer 3 switch to a router, another Layer 3 switch, or when you need point-to-point Layer 3 connections. They eliminate the need for VLANs on inter-switch links.

Routed Port vs SVI (Switch Virtual Interface)

Method What It Is When to Use
Routed Port Physical port with IP address Point-to-point connections to routers/switches
SVI Virtual interface for a VLAN Inter-VLAN routing for multiple devices

My Lab Topology

One multilayer switch (Layer 3 switch) with two computers. Computer A in VLAN 10, Computer B in default VLAN 1. Switch port Fa0/2 configured as routed port connecting VLAN 10 to VLAN 1.

Network Diagram

Topology:
  • Computer A (VLAN 10) → Switch Fa0/1
  • Computer B (VLAN 1) → Switch Fa0/3
  • Switch Fa0/2 = Routed Port (connects the two VLANs)

Device Configuration

Device Interface IP Address Subnet Mask VLAN
Computer A NIC 192.168.10.2 255.255.255.0 10
Computer B NIC 192.168.10.3 255.255.255.0 1 (default)
Switch A Fa0/1 — — 10 (access)
Switch A Fa0/2 192.168.10.254 255.255.255.0 Routed Port
Switch A Fa0/3 — — 1 (default)

Configuration Steps

Step 1 — Create VLAN 10 and Configure Access Port

Cisco IOS — Switch A 
enable
configure terminal

! Create VLAN 10
vlan 10
 exit

! Configure Fa0/1 as access port in VLAN 10
interface fastethernet0/1
 switchport mode access
 switchport access vlan 10
 no shutdown
 exit

Step 2 — Configure Fa0/2 as Routed Port

Convert to Routed Port 
! Configure Fa0/2 as Layer 3 routed port
interface fastethernet0/2
 no switchport
 ip address 192.168.10.254 255.255.255.0
 no shutdown
 exit
Important: The no switchport command converts the port from Layer 2 to Layer 3. After this, you can assign an IP address directly to the interface, and it will no longer be associated with any VLAN.

Step 3 — Enable IP Routing on Switch

Enable Layer 3 Routing 
! Enable IP routing globally
ip routing

! Save configuration
end
write memory
ip routing Command: This enables routing on the Layer 3 switch. Without this command, the switch won't forward packets between different subnets/VLANs even if routed ports or SVIs are configured.

Step 4 — Verify Configuration

Verification Commands 
! Check if interface is routed port (not switchport)
show interface fa0/2 switchport

! Should show: "Switchport: Disabled" or "not a switchport"

! Check IP interface status
show ip interface brief

! Check routing table
show ip route

! Verify VLANs
show vlan brief

Step 5 — Test Connectivity

From Computer A (VLAN 10) 
! Ping default gateway (routed port)
ping 192.168.10.254

! Ping Computer B in different VLAN
ping 192.168.10.3

! Expected: Both pings should succeed
After configuration, Computer A (VLAN 10) successfully pinged Computer B (VLAN 1) at 192.168.10.3. The switch was routing traffic between VLANs at Layer 3 without needing an external router!

Key Takeaways

What I Understood

  • no switchport converts a Layer 2 port to Layer 3
  • Routed ports can have IP addresses assigned directly
  • ip routing must be enabled for inter-VLAN routing
  • Routed ports are NOT associated with any VLAN
  • Layer 3 switches eliminate need for external routers in some designs
  • Routed ports are typically used for point-to-point connections
  • show interface switchport shows if port is routed

When to Use Routed Ports

Use Routed Port For Use SVI For
Point-to-point links to routers Routing between multiple devices in VLANs
Connecting Layer 3 switches together Default gateway for end users
WAN links When many devices need routing
Simple two-device connections Campus network inter-VLAN routing

Common Mistakes

  • Forgot ip routing: Configured routed port but forgot global ip routing command — routing didn't work
  • Tried to assign VLAN: Attempted switchport access vlan 10 after no switchport — got error
  • Wrong subnet on computers: Computer B had wrong subnet initially — couldn't ping across VLANs
