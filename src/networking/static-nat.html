What I Learned

I configured Static NAT between two routers in Packet Tracer to understand how private IP addresses get mapped to public ones. This was my first time seeing NAT actually work.

The concept is simple: one private IP permanently maps to one public IP. But seeing it translate packets in real-time made me realize why servers need this.

What is Static NAT?

Static NAT creates a permanent one-to-one mapping between a private IP and a public IP.

Example: My internal server at 10.0.0.2 is always translated to 50.0.0.3 when communicating outside.

Use Static NAT when an internal device (like a web server) needs to be reachable from another network using a consistent public IP.

My Lab Topology

PC1 → Router A → (Serial Link) → Router B → PC2

Device Interface IP Address
PC1—10.0.0.2
Router AFa0/010.0.0.1
Router AS0/0/020.0.0.1
Router BS0/0/020.0.0.2
Router BFa0/0192.168.0.1
PC2—192.168.0.2

NAT Mappings I Created

  • Router A: 10.0.0.2 → 50.0.0.3
  • Router B: 192.168.0.2 → 30.0.0.1

Configuration Commands

Router A

Cisco IOS 
enable
conf t
interface fastethernet0/0
 ip address 10.0.0.1 255.255.255.0
 ip nat inside
 no shutdown
interface serial0/0/0
 ip address 20.0.0.1 255.255.255.0
 ip nat outside
 no shutdown
ip nat inside source static 10.0.0.2 50.0.0.3

Router B

Cisco IOS 
enable
conf t
interface fastethernet0/0
 ip address 192.168.0.1 255.255.255.0
 ip nat inside
 no shutdown
interface serial0/0/0
 ip address 20.0.0.2 255.255.255.0
 ip nat outside
 no shutdown
ip nat inside source static 192.168.0.2 30.0.0.1

Key Takeaways

What I Understood

  • Inside interface = connects to private network
  • Outside interface = connects to public network
  • Static mapping is permanent — doesn't change
  • Perfect for servers that need consistent public IPs
  • Each mapping uses one public IP address

When I'd Use This

Static NAT is ideal when:

  • Running a web server that external users need to access
  • Hosting services that require a fixed IP address
  • Need incoming connections from the internet
