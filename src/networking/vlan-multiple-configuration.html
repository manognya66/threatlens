What I Learned

I created three separate VLANs on a single switch and saw real network segmentation in action. Computers in VLAN 100 couldn't ping computers in VLAN 200 or 300 — complete isolation at Layer 2, even though they're all on the same physical switch.

The interface range command was a game-changer for configuring multiple ports at once. Instead of configuring 10 ports individually, I grouped them and applied VLAN settings in one command.

VLAN Segmentation

VLAN Segmentation divides a physical network into multiple logical networks. Each VLAN is a separate broadcast domain, providing isolation, security, and traffic control without requiring separate physical switches.

Why Segment with Multiple VLANs?

Real-World Example: A company might use:
  • VLAN 100 → Management/Admin
  • VLAN 200 → Employees
  • VLAN 300 → Guest WiFi
Each department is isolated, preventing guests from accessing employee resources.

Interface Range Command

The interface range command configures multiple interfaces simultaneously:

Interface Range Examples 
! Configure Fa0/10 through Fa0/20
interface range fa0/10-20

! Configure Fa0/1, Fa0/5, and Fa0/10
interface range fa0/1, fa0/5, fa0/10

! Configure Fa0/10-20 AND Fa0/30
interface range fa0/10-20, fa0/30

My Lab Topology

One switch connecting multiple computers across three VLANs. Different port ranges assigned to different VLANs for department segmentation.

VLAN Design

VLAN ID VLAN Name Ports Purpose
100 Management Fa0/10-20, Fa0/30 Admin & IT staff
200 Employees Fa0/1-9 Regular employees
300 Guests Fa0/21-24 Guest access

Expected Behavior

Communication Rules:
  • ✅ Devices in VLAN 100 can ping each other
  • ✅ Devices in VLAN 200 can ping each other
  • ✅ Devices in VLAN 300 can ping each other
  • ❌ VLAN 100 cannot ping VLAN 200 (isolated)
  • ❌ VLAN 200 cannot ping VLAN 300 (isolated)
  • ❌ VLAN 100 cannot ping VLAN 300 (isolated)

Configuration Steps

Step 1 — Create All VLANs

Cisco IOS — Switch Configuration 
enable
configure terminal

! Create VLAN 100
vlan 100
 name Management
 exit

! Create VLAN 200
vlan 200
 name Employees
 exit

! Create VLAN 300
vlan 300
 name Guests
 exit

Step 2 — Assign Port Ranges to VLANs

VLAN 100 (Management) — Fa0/10-20 and Fa0/30

VLAN 100 Configuration 
! Configure range Fa0/10 through Fa0/20
interface range fa0/10-20
 switchport mode access
 switchport access vlan 100
 no shutdown
 exit

! Configure Fa0/30
interface fa0/30
 switchport mode access
 switchport access vlan 100
 no shutdown
 exit

VLAN 200 (Employees) — Fa0/1-9

VLAN 200 Configuration 
! Configure range Fa0/1 through Fa0/9
interface range fa0/1-9
 switchport mode access
 switchport access vlan 200
 no shutdown
 exit

VLAN 300 (Guests) — Fa0/21-24

VLAN 300 Configuration 
! Configure range Fa0/21 through Fa0/24
interface range fa0/21-24
 switchport mode access
 switchport access vlan 300
 no shutdown
 exit

! Save configuration
end
write memory
Pro Tip: Using interface range saved me tons of time. Instead of typing 11 separate interface configurations for VLAN 100, I did it in two commands!

Verification

! Verify specific interface

Step 4 — Test VLAN Isolation

Testing Results:
  • Computer on Fa0/10 (VLAN 100) ✅ pinged computer on Fa0/15 (VLAN 100) — Success
  • Computer on Fa0/10 (VLAN 100) ❌ pinged computer on Fa0/5 (VLAN 200) — Failed (isolated)
  • Computer on Fa0/22 (VLAN 300) ✅ pinged computer on Fa0/23 (VLAN 300) — Success
This proved VLANs were properly isolating traffic!

Optional — Bulk Configuration Tips

Advanced Interface Range 
! Configure multiple non-contiguous ranges at once
interface range fa0/1-9, fa0/21-24
 description Employee and Guest ports
 spanning-tree portfast
 no shutdown
 exit

! Apply settings to ALL FastEthernet ports
interface range fa0/1-24
 switchport mode access
 exit

Key Takeaways

What I Understood

  • Multiple VLANs on one switch provide complete Layer 2 isolation
  • interface range command drastically speeds up configuration
  • Each VLAN is a separate broadcast domain
  • Devices in different VLANs cannot communicate without a router
  • VLAN names help identify purpose but don't affect functionality
  • Port ranges can be non-contiguous (Fa0/10-20, Fa0/30)
  • show vlan brief shows all VLANs and their member ports

VLAN Design Best Practices

Practice Reason
Plan VLAN IDs in ranges 100-199 = Management, 200-299 = Users, etc.
Always name VLANs Easier troubleshooting and documentation
Document port assignments Know which ports belong to which VLAN
Use interface range Faster configuration, fewer errors
Shutdown unused ports Security — prevent unauthorized connections

Interface Range Syntax

Command What It Does
interface range fa0/1-10 Fa0/1 through Fa0/10 (continuous)
interface range fa0/1, fa0/5 Only Fa0/1 and Fa0/5 (specific ports)
interface range fa0/1-10, fa0/20 Fa0/1-10 plus Fa0/20
interface range fa0/1-24 All FastEthernet ports

Common Mistakes

  • Wrong range syntax: Used spaces instead of hyphens — fa0/1 - 10 instead of fa0/1-10
  • Forgot to create VLAN first: Assigned ports to VLAN 100 before creating it — Cisco auto-created it but best practice is explicit creation
  • Tested within same VLAN: Initially tested ping between two computers in VLAN 100 and thought isolation failed — needed to test across VLANs
